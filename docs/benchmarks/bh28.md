# BH28 Benchmark

Comprehensive evaluation of QME backends on the BH28 database of chemical reaction barrier heights.

## Overview

The **BH28 Benchmark** provides a unified evaluation of all QME backends on the BH28 database of chemical reaction barrier heights. This benchmark tests the accuracy of machine learning potentials for predicting reaction barriers across diverse chemical reactions.

## What the Benchmark Does

1. **Optimizes reactant minima** using various QME ML backends
2. **Optimizes transition states** using SELLA optimizer
3. **Calculates barrier heights** from optimized structures
4. **Compares accuracy** against reference values from high-level quantum chemistry
5. **Provides performance analysis** and backend recommendations

## Usage

### Running the Benchmark

```bash
# Navigate to benchmark directory
cd examples/bh28_benchmark

# Run comprehensive benchmark on all available backends
python bh28_benchmark.py

# Quick test with representative subset of reactions
python bh28_benchmark.py --quick

# Test specific backends
python bh28_benchmark.py --backends mock so3lr uma

# Test specific reactions
python bh28_benchmark.py --reactions BHDIV_3 PXBH_2 CADBH_1

# Analysis only (load existing results)
python bh28_benchmark.py --analyze

python bh28_benchmark.py --quick
```

### Command Line Options

- `--backends`: Comma-separated list of backends to test
- `--reactions`: Comma-separated list of specific reactions to test
- `--quick`: Run on a representative subset of reactions for faster testing
- `--analyze`: Load and analyze existing results without running new calculations
- `--device`: Device to use for calculations ("cpu" or "cuda")
- `--output`: Output file for results (default: benchmark_results/bh28_benchmark_results.json)

## BH28 Dataset

The benchmark uses the BH28 dataset containing 28 diverse chemical reactions with reference barrier heights from CCSDT(Q)/CBS calculations:

### Reaction Types

- **PXBH**: Proton exchange barriers (H-abstraction reactions)
- **BHPERI**: Pericyclic reactions (Diels-Alder, Cope rearrangement, etc.)
- **CADBH**: Cyclization and addition barriers (bimolecular: small molecule + C2H4)
- **CRBH**: Cyclic radical barriers
- **BHDIV**: Diverse barrier heights (various reaction types)

### Dataset Structure

Each reaction includes:
- Reactant minimum structure (`*_min.xyz`)
- Transition state structure (`*_ts.xyz`)
- Reference barrier height from high-level quantum chemistry

### Example Reactions

| Reaction | Type | Reference Barrier (eV) | Description |
|----------|------|----------------------|-------------|
| BHDIV_3 | Diverse | 1.52 | H + CH₄ → CH₃ + H₂ |
| PXBH_2 | Proton Exchange | 0.48 | OH + CH₄ → H₂O + CH₃ |
| BHPERI_1 | Pericyclic | 1.31 | Diels-Alder reaction |
| CADBH_1 | Addition | 0.62 | CH₃ + C₂H₄ → C₃H₇ |

## Output and Results

### Result Files

Results are saved to `benchmark_results/bh28_benchmark_results.json` and include:

- **Optimized structures** for all reactants and transition states
- **Energy values** and calculated barrier heights
- **Performance metrics** (MAE, RMSE, timing statistics)
- **Detailed analysis** comparing backend accuracy
- **Success rates** for optimization convergence

### Performance Metrics

- **Success Rate**: Percentage of reactions where optimization converged
- **MAE (Mean Absolute Error)**: Average absolute difference from reference barrier heights
- **RMSE (Root Mean Square Error)**: Root mean square difference from reference
- **Timing**: Average time per reaction optimization

## Interpreting Results

### Accuracy Guidelines

- **Excellent**: MAE < 0.15 eV
- **Good**: MAE 0.15-0.25 eV
- **Acceptable**: MAE 0.25-0.40 eV
- **Poor**: MAE > 0.40 eV

### Backend Recommendations

Based on typical BH28 results:

- **High Accuracy**: AIMNet2, MACE, TorchSim typically show lowest errors
- **Speed**: TorchSim backends often provide best performance
- **Reliability**: AIMNet2 and MACE show good convergence rates
- **General Purpose**: UMA provides good balance for diverse systems

### Chemical Insights

The BH28 benchmark reveals backend strengths for different reaction types:

- **Hydrogen abstraction (PXBH)**: Generally well-predicted by all backends
- **Pericyclic reactions (BHPERI)**: May show larger errors, require careful backend selection
- **Addition reactions (CADBH)**: Usually well-handled by modern ML potentials
- **Complex mechanisms (BHDIV)**: Most challenging, best for distinguishing backend quality

## Requirements

### Software Dependencies
- **QME package**: Base QME installation
- **ML backends**: At least one of: UMA (`fairchem-core`), SO3LR, AIMNet2, MACE
- **Transition states**: SELLA optimizer (`pip install sella`) - recommended for proper TS optimization
- **Python environment**: Python 3.10+ recommended

### Hardware Requirements
- **CPU**: Multi-core recommended for parallel optimization
- **Memory**: 8GB+ RAM for larger molecules
- **GPU**: Optional but recommended for CUDA-compatible backends

## Advanced Usage

### Custom Analysis
```python
import json
import numpy as np

# Load results
with open('benchmark_results/bh28_benchmark_results.json', 'r') as f:
    results = json.load(f)

# Custom analysis
for backend, data in results['backends'].items():
    barriers_calc = [r['barrier_height'] for r in data['reactions'].values()]
    barriers_ref = [r['reference_barrier'] for r in data['reactions'].values()]

    mae = np.mean(np.abs(np.array(barriers_calc) - np.array(barriers_ref)))
    print(f"{backend}: MAE = {mae:.3f} eV")
```

### Batch Processing
```bash
# Run multiple benchmarks with different settings
for backend in uma aimnet2 mace; do
    python bh28_benchmark.py --backends $backend --output results_${backend}.json
done
```

## Troubleshooting

### Common Issues

**No backends available**:
```
Error: No ML backends available for benchmarking
```
**Solution**: Install at least one backend: `pip install qme-ml-ml[uma]`

**Optimization failures**:
```
Warning: Optimization failed for reaction BHDIV_3 with backend uma
```
**Solution**: Normal for some difficult reactions; results are reported but don't stop benchmark

**Memory issues**:
```
CUDA out of memory
```
**Solution**: Use `--device cpu` or reduce system size

### Performance Tips

1. **Use GPU acceleration**: `--device cuda` for compatible backends
2. **Parallel execution**: Run different backends on separate cores
3. **Quick testing**: Use `--quick` flag for initial validation
4. **Incremental analysis**: Use `--analyze` to examine existing results

For more help, see the [Troubleshooting Guide](../reference/troubleshooting.md).
