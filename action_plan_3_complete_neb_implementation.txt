## Unified TS Search Design Philosophy

### Smart Default Strategy

**Primary Design Goal**: Maximize usability with intelligent defaults while maintaining flexibility for expert users.

**Recommended CLI Command**: `qme find-ts reactant.xyz product.xyz`

**Method Selection Logic:**
1. **Default ('auto')**: Geodesic midpoint + single TS optimization
   - **Why**: Fastest, works for 80% of cases, leverages QME's geodesic interpolation
   - **When it fails**: Complex reaction coordinates, multiple transition states
   - **Fallback**: User can specify `--method neb` if needed

2. **User specifies images**: Automatically switches to NEB-based approach
   - `--n-images 1`: Geodesic midpoint approach (same as auto)
   - `--n-images >1`: Climbing Image NEB with specified images
   - **Rationale**: User intent is clear when they specify multiple images

3. **Explicit method selection**: Override defaults for expert control
   - `--method direct`: Force geodesic midpoint (fast)
   - `--method neb/cineb`: Force NEB-based approach (thorough)

### Usability Principles

**Default Behavior Optimization:**
- **Speed first**: Default to fastest method that works for most cases
- **Progressive complexity**: Easy to escalate to more thorough methods
- **Automatic verification**: Default to frequency analysis verification
- **Clear feedback**: Report which method was used and why

**Image Count Auto-Selection Logic:**
- **n_images=None (default)**: 
  - Method 'auto'/'direct': Uses 1 image (geodesic midpoint)
  - Method 'neb'/'cineb': Uses 7 images (optimal balance)
- **n_images=1**: Forces geodesic midpoint approach regardless of method
- **n_images>1**: Forces NEB-based approach with specified count
- **Rationale**: Most users don't need to think about image counts; experts can override

**Examples of Usage Patterns:**
```bash
# Quick TS search (recommended for most users)
qme find-ts reactant.xyz product.xyz -o ts.xyz

# Force thorough NEB approach
qme find-ts reactant.xyz product.xyz --method neb -o ts.xyz

# Complete validation workflow
qme find-ts reactant.xyz product.xyz --verify-ts --validate-irc -o ts.xyz

# Expert control over image count (auto-selects NEB method)
qme find-ts reactant.xyz product.xyz --n-images 11 -o ts.xyz

# Fast single-point approach even with NEB method specified
qme find-ts reactant.xyz product.xyz --method neb --n-images 1 -o ts.xyz
```

        effective_method = 'direct'
        effective_n_images = n_images or 1
    elif method == 'direct':
        effective_method = 'direct' 
        effective_n_images = n_images or 1
    elif method in ['neb', 'cineb']:

        # Fast approach: geodesic midpoint + TS optimization

## 2. Mathematical Implementation Details

### 2.1 Core NEB Algorithm Implementation

**Spring Force Calculation:**
```python
def calculate_spring_forces(self, i, tangent):
    """
    Calculate spring forces for image i.
    F_spring_i = k * (|R_{i+1} - R_i| - |R_i - R_{i-1}|) * τ_i
    """
    if i == 0 or i == len(self.images) - 1:
        return np.zeros_like(self.images[i].positions.flatten())
    
    R_prev = self.images[i-1].positions.flatten()
    R_curr = self.images[i].positions.flatten()
    R_next = self.images[i+1].positions.flatten()
    
    dist_next = np.linalg.norm(R_next - R_curr)
    dist_prev = np.linalg.norm(R_curr - R_prev)
    
    spring_force = self.k[i] * (dist_next - dist_prev) * tangent
    return spring_force
```

**Improved Tangent Calculation (Henkelman-Jonsson):**
```python
def calculate_improved_tangent(self, i, energies):
    """
    Calculate improved tangent vector according to Henkelman & Jonsson.
    Handles cases where image is at maximum, minimum, or inflection point.
    """
    if i == 0:
        tangent = self.images[1].positions - self.images[0].positions
    elif i == len(self.images) - 1:
        tangent = self.images[-1].positions - self.images[-2].positions
    else:
        R_prev = self.images[i-1].positions.flatten()
        R_curr = self.images[i].positions.flatten()
        R_next = self.images[i+1].positions.flatten()
        
        tau_plus = R_next - R_curr
        tau_minus = R_curr - R_prev
        
        E_prev = energies[i-1]
        E_curr = energies[i]
        E_next = energies[i+1]
        
        # Determine tangent based on energy landscape
        if E_next > E_curr > E_prev:
            tangent = tau_plus
        elif E_next < E_curr < E_prev:
            tangent = tau_minus
        else:
            # At extremum - use weighted combination
            dE_max = max(abs(E_next - E_curr), abs(E_prev - E_curr))
            dE_min = min(abs(E_next - E_curr), abs(E_prev - E_curr))
            
            if E_next > E_prev:
                tangent = tau_plus * dE_max + tau_minus * dE_min
            else:
                tangent = tau_plus * dE_min + tau_minus * dE_max
    
    # Normalize tangent
    tangent_norm = np.linalg.norm(tangent)
    if tangent_norm > 1e-10:
        tangent = tangent / tangent_norm
    
    return tangent
```

**Climbing Image Force Modification:**
```python
def apply_climbing_image_forces(self, i, forces, tangent):
    """
    Modify forces for climbing image by reversing parallel component.
    F_climb = F_perp - F_parallel = F - 2(F·τ)τ
    """
    parallel_component = np.dot(forces, tangent) * tangent
    climbing_forces = forces - 2 * parallel_component
    return climbing_forces
```

### 2.2 Variable Spring Constants

**Energy-dependent spring constants:**
```python
def update_variable_springs(self, energies):
    """
    Update spring constants based on local energy landscape.
    Higher spring constants in high energy regions.
    """
    E_min = np.min(energies)
    E_max = np.max(energies)
    E_ref = E_min + 0.85 * (E_max - E_min)
    
    for i in range(len(self.k)):
        # Spring i connects images i and i+1
        E_local = max(energies[i], energies[i+1])
        
        if E_local < E_ref:
            self.k[i] = self.k_min
        else:
            ratio = (E_max - E_local) / (E_max - E_ref)
            self.k[i] = self.k_max - self.delta_k * ratio
```

**Image Dependent Pair Potential interpolation:**

def idpp_interpolate(self, initial, final, n_images):
    """
    Use IDPP method for better initial path guess.
    Creates chemically reasonable intermediate structures.
    """
    # Create initial linear interpolation
    linear_path = self.linear_interpolate(initial, final, n_images)
    
    # Set up IDPP potential (harmonic in pairwise distances)
    for image in linear_path:
        image.calc = IDPPCalculator()
    
    # Optimize using NEB with IDPP potential
    idpp_neb = QME_NEB(linear_path, k=1.0)
    idpp_neb.optimize(optimizer='FIRE', fmax=0.1, steps=1000)
    
    # Remove IDPP calculators and return improved path
    for image in linear_path:
        image.calc = None
    
    return linear_path
```

@click.argument("reactant_file", type=click.Path(exists=True))

@click.option("--output", "-o", help="Output file for transition state structure")
@click.option("--n-images", "-n", default=None, type=int,

              type=click.Choice(['auto', 'direct', 'neb', 'cineb']),

              help="Verify TS with frequency analysis")
@click.option("--validate-irc", is_flag=True, default=False,
              help="Validate TS connects given minima with IRC")
@add_common_options(core_options)
@add_common_options(optimization_options)
def find_ts(reactant_file, product_file, output, n_images, method, 
           spring_constant, verify_ts, validate_irc, **kwargs):
    """
    Find transition state connecting two minima.
    
    INTELLIGENT DEFAULTS:
    - method='auto': Uses geodesic interpolation + single image TS optimization
    - n_images=None: Auto-selects 1 for 'direct', 5-9 for NEB methods
    - Automatically verifies TS with frequency analysis
    - Optional IRC validation to confirm TS connects the given minima
    
    EXAMPLES:
    # Quick TS search (recommended for most cases)
    qme find-ts reactant.xyz product.xyz -o ts.xyz
    
    # Force NEB with specific number of images  
    qme find-ts reactant.xyz product.xyz --method neb --n-images 7
    
    # Complete validation (TS + frequencies + IRC validation)
    qme find-ts reactant.xyz product.xyz --verify-ts --validate-irc
    """
    
    if kwargs.get('verbose'):
        click.echo(f"Starting intelligent TS search...")
        click.echo(f"Method: {method}")
        if n_images:
            click.echo(f"Images: {n_images}")
        else:
            click.echo("Images: auto-select")
    
    try:
        # Setup QME optimizer
        qme, _ = _setup_optimization(
            None, kwargs.get('backend'), kwargs.get('model'), 
            kwargs.get('model_path'), kwargs.get('device'), None, 
            kwargs.get('verbose')
        )
        
        # Load reactant and product structures
        reactant = read(reactant_file)
        product = read(product_file)
        
        if kwargs.get('verbose'):
            click.echo(f"Loaded reactant: {len(reactant)} atoms")
            click.echo(f"Loaded product: {len(product)} atoms")
        
        # Run intelligent TS search
        results = qme.find_transition_state_smart(
            reactant=reactant,
            product=product,
            method=method,
            n_images=n_images,
            verify_ts=verify_ts,
            validate_irc=validate_irc,
            spring_constant=spring_constant,
            **kwargs
        )
        
        # Handle results
        if results['converged']:
            click.echo("✓ Transition state search converged!")
            click.echo(f"Method used: {results['method']}")
            
            if 'frequency_analysis' in results:
                if results['is_valid_ts']:
                    click.echo("✓ TS verified: exactly one imaginary frequency")
                else:
                    click.echo("⚠ Warning: TS verification failed (frequency analysis)")
            
            if 'irc_validation' in results:
                if results['irc_validation']['connects_endpoints']:
                    click.echo("✓ IRC validation: TS connects given minima")
                else:
                    click.echo("⚠ Warning: IRC validation failed")
            
            # Save TS structure
            if output:
                write(output, results['atoms'])
                click.echo(f"Transition state saved to: {output}")
            
        else:
            click.echo("✗ Transition state search failed to converge")
            click.echo(f"Method attempted: {results.get('method', method)}")
            
            # Suggest alternative approach
            if method == 'auto' or method == 'direct':
                click.echo("💡 Suggestion: Try --method neb for more thorough search")
            elif n_images and n_images < 7:
                click.echo("💡 Suggestion: Try more images (--n-images 9)")
        
    except Exception as e:
        click.echo(f"Error during TS search: {e}", err=True)
        sys.exit(1)

def setup_initial_path_with_geodesic(self, reactant, product, n_images):
    """
    Use QME's geodesic interpolation for initial NEB path.
    
    This provides much better initial paths than linear interpolation
    by preserving bond lengths and chemical reasonableness.
    """
    from .reaction import Reaction
    
    # Create temporary Reaction object to access geodesic interpolation
    reaction = Reaction(reactant, product)
    
    # Use geodesic interpolation instead of linear
    path_geometries = reaction.interpolate_path(
        npoints=n_images,
        method='geodesic',
        optimize_path=False  # Don't optimize yet, NEB will do that
    )
    
    return path_geometries

def create_neb_from_endpoints(self, reactant, product, n_images=9, **neb_kwargs):
    """

### 6.1 Parallel Force Calculations

```python
def calculate_forces_parallel(self, images):
    """
    Calculate forces for multiple images in parallel.
    Useful for independent image force evaluations.
    """
    if self.parallel and len(images) > 1:
        # Use multiprocessing or threading for parallel evaluation
        from concurrent.futures import ProcessPoolExecutor
        
        with ProcessPoolExecutor() as executor:
            force_futures = []
            for image in images:
                future = executor.submit(image.get_forces)
                force_futures.append(future)
            
            forces = [future.result() for future in force_futures]
        return forces
    else:
        # Sequential calculation
        return [image.get_forces() for image in images]
```
